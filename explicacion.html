📋 Explicación del Carrito de Compras

1. Estructura de archivos

Se agregaron o modificaron tres archivos sobre el proyecto original:

  • index.html — se agregó una línea al final para cargar el script
  • js/carrito.js — archivo nuevo con toda la lógica del carrito
  • pages/carrito.html — página del carrito reemplazada con diseño y funcionalidad

2. Array de productos

En carrito.js se define un array llamado productos con los tres servicios disponibles. Cada producto tiene: id, nombre, descripcion, precio y storage (indica dónde se guarda).

const productos = [
  { id: 1, nombre: "Transporte Logístico",   precio: 150000, storage: "local"   },
  { id: 2, nombre: "Venta de Pasajes",        precio: 80000,  storage: "session" },
  { id: 3, nombre: "Servicio de Mantenimiento", precio: 200000, storage: "local" }
];

3. Eventos en los botones de compra

Al cargar la página, el script busca todos los elementos con clase .btn-compra y les asigna un evento click a cada uno. El atributo preventDefault() evita que el enlace <a> navegue a otra página.

document.addEventListener("DOMContentLoaded", function() {
  var botones = document.querySelectorAll(".btn-compra");
  botones.forEach(function(boton, indice) {
    boton.addEventListener("click", function(evento) {
      evento.preventDefault();
      mostrarOpciones(indice + 1);
    });
  });
});

4. Ventana de confirmación

Al hacer clic en un botón se ejecuta mostrarOpciones(), que muestra un confirm() con el nombre, descripción y precio del producto. Si el usuario confirma, se llama a agregarAlCarrito().

function mostrarOpciones(idProducto) {
  const producto = productos.find(function(p) { return p.id === idProducto; });
  const confirmar = confirm(
    producto.nombre + "\n" + producto.descripcion + "\nPrecio: $" + producto.precio
  );
  if (confirmar) { agregarAlCarrito(producto); }
}

5. localStorage vs sessionStorage

Dependiendo del campo storage del producto, se guarda en uno u otro:

Botón Producto Dónde se guarda ¿Persiste?
Botón 1 Transporte Logístico localStorage ✅ Sí, aunque cierres el navegador
Botón 2 Venta de Pasajes sessionStorage ❌ Se borra al cerrar la pestaña
Botón 3 Servicio de Mantenimiento localStorage ✅ Sí, aunque cierres el navegador 
// Guardar en localStorage
localStorage.setItem("carritoLocal", JSON.stringify(carritoLocal));

// Guardar en sessionStorage
sessionStorage.setItem("carritoSession", JSON.stringify(carritoSession));

Los datos se guardan como texto en formato JSON usando JSON.stringify(). Para leerlos se usa JSON.parse().

6. Página del carrito

La página carrito.html se encarga de:

  • Leer los datos guardados en localStorage y sessionStorage
  • Mostrar cada producto con su nombre, descripción, precio y botón Eliminar
  • Calcular y mostrar el subtotal de cada storage y el total general
  • Permitir vaciar todo el carrito con el botón Vaciar carrito

La función principal es renderCarrito(), que se ejecuta al cargar la página y cada vez que se elimina un producto.

function renderCarrito() {
  var carritoLocal   = JSON.parse(localStorage.getItem("carritoLocal"))   || [];
  var carritoSession = JSON.parse(sessionStorage.getItem("carritoSession")) || [];
  // ... recorre cada array, construye el HTML y suma los precios
}

7. Eliminar un producto

Cada tarjeta de producto tiene un botón Eliminar que llama a eliminarLocal(i) o eliminarSession(i) según corresponda, donde i es la posición del producto en el array. Se usa splice() para quitarlo y se vuelve a guardar el array actualizado.

function eliminarLocal(i) {
  var carrito = JSON.parse(localStorage.getItem("carritoLocal")) || [];
  carrito.splice(i, 1);          // elimina 1 elemento en la posición i
  localStorage.setItem("carritoLocal", JSON.stringify(carrito));
  renderCarrito();               // actualiza la pantalla
}
